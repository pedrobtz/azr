## Package Overview

`azr` is an R package that implements an Azure OAuth 2.0 credential chain for seamless authentication to Azure services. It is built on top of `httr2`'s OAuth framework and sequentially attempts authentication methods until one succeeds. It also provides a base HTTP client layer (`api_client`, `api_resource`, `api_service`) for building Azure service-specific API wrappers.

## Architecture

### Core Components

**Credential System**: R6 class hierarchy for Azure OAuth 2.0 authentication
- `Credential` (`R/credential.R`) — Abstract base class. All credential types inherit from it. Locks all public fields after `initialize()` to prevent mutation.
- `DefaultCredential` (`R/default-credential.R`) — Lazily initialises a credential provider on first access using the default credential chain.
- `credential_chain()` / `default_credential_chain()` — Ordered list of credential classes to try in sequence.
- `get_credential_provider()` — Walks the chain and returns the first credential that successfully gets a token.
- `get_token_provider()`, `get_request_authorizer()`, `get_token()`, `get_credential_auth()` — Convenience wrappers around `get_credential_provider()`.

**Credential Implementations** (each in `R/credential-*.R`):
- `ClientSecretCredential` — Non-interactive; uses `AZURE_CLIENT_ID` + `AZURE_CLIENT_SECRET` + `AZURE_TENANT_ID`.
- `AuthCodeCredential` — Interactive browser-based Authorization Code flow (requires `httpuv`).
- `DeviceCodeCredential` — Interactive Device Code flow; displays a code to enter at `microsoft.com/devicelogin`.
- `AzureCLICredential` — Delegates to the `az` CLI for token acquisition.
- `RefreshTokenCredential` — Uses a stored refresh token (`AZURE_REFRESH_TOKEN`).
- `CachedTokenCredential` — Returns a cached token from disk or memory.

**HTTP Client Layer**: Base classes for building Azure API wrappers
- `api_client` (`R/api-client.R`) — Base HTTP client with authentication, retry/backoff, and logging. `.fetch()` is the main request method; supports path interpolation via `rlang::englue()`.
- `api_resource` (`R/api-resource.R`) — Extends `api_client` with a fixed path prefix (e.g., `"v1.0"`, `"beta"`).
- `api_service` (`R/api-services.R`) — Base class for service-specific wrappers; holds a client and named endpoint resources.

**Pre-built Service Clients**:
- `AzureGraphClient` (`R/azure-graph-client.R`) — Microsoft Graph API client.
- `AzureStorageClient` (`R/azure-storage-client.R`) — Azure Blob Storage client.

**Defaults and Constants** (`R/defaults.R`, `R/constants.R`):
- Environment variables read: `AZURE_TENANT_ID`, `AZURE_CLIENT_ID`, `AZURE_CLIENT_SECRET`, `AZURE_AUTHORITY_HOST`, `AZURE_REFRESH_TOKEN`, `AZURE_CONFIG_DIR`.
- Built-in scopes in `azure_scopes`: `azure_arm`, `azure_graph`, `azure_storage`, `azure_key_vault`, `azure_openai`.
- Built-in authority hosts in `azure_authority_hosts`: `azure_public_cloud`, `azure_government`, `azure_china`.
- Default client ID is Microsoft's public Azure CLI client (`04b07795-8ddb-461a-bbee-02f9e1bf7b46`).

### Key Design Patterns

**R6 with locked bindings**: All credential and client objects lock their public fields after `initialize()` using `lockBinding()`. Do not attempt to mutate fields after construction.

**Credential chain pattern**: `credential_chain()` captures unevaluated expressions (via `rlang::enquos()`). `get_credential_provider()` evaluates each lazily: if the element is an R6 class it calls `new_instance()` (which discovers matching constructor arguments from the caller environment); if it is already an R6 instance it is used directly.

**`new_instance()` introspection**: Reads the `initialize()` argument names of an R6 class and pulls matching names from the calling environment. This allows `get_credential_provider()` to forward `scope`, `tenant_id`, etc. automatically.

**Response handler**: `api_client` applies a `response_handler` callback to parsed JSON responses. The default (`default_response_handler()`) converts `data.frame` results to `data.table` when the `data.table` package is available.

**Path interpolation**: `api_client$.fetch()` accepts a `path` string and named `...` arguments; the path is interpolated using `rlang::englue()` so you can write `"/subscriptions/{subscription_id}/resourceGroups"`.
